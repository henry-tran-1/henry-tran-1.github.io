The CSS Box Model

The box model is important to know about in web design because all HTML elements can be considered boxes.
Each element is a box that consists of 4 different layers. These layers in order are:

  • Content (blue box):
    This is the area where your content is displayed, e.g. texts and images.
  • Padding (green box):
    This is the transparent space between the content area and the border.
    Any background colour applied to the element will also display behind the padding space.
  • Border (yellow box):
    This is the border that wraps around the content and padding space.
    It is situated between the padding and the margin.
  • Margin (orange box):
    The margin is the last layer and wraps around the other three layers.
    It is an invisible space around the box, that will push away other elements.
    This has no background.

There are also two types of box models:

  • Standard box model: width/height attributes correspond to the content box, and then padding/borders are added on top. This is the default.
  • Alternate box model: width/height attributes correspond to the total size of the content box, the padding box and the border boxes all added together.

How do the margins of two elements/boxes interact with each other?

If there are two boxes, each with a margin of 50px, they would overlap as shown in the image to the right: they are separated by only 50px of space. The margins do not add up to make 100px of space.
This is because the elements can be as close to each other as they like, as long as there's enough space between them to satisfy each of their margin requirements. If there are two boxes with different margins, the space between them would be equal to the largest margin of the two.
I like to think of the margin attribute like a person's personal space. If two people both had a personal space of 2 metres, they would only have to be 2 metres away from each other to make both people happy.
